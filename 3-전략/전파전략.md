# HypoLoop 전파 전략

## 0단계 — 홈부터 (1주)

GitHub Pages에 랜딩 페이지와 시각화 자료를 올려 canonical URL을 확보합니다 (예: hypoloop.dev 또는 devkoan.github.io/hypoloop). 모든 채널이 이 주소로 수렴해야 검색 자산이 쌓입니다. "가설순환학습"은 검색 결과가 비어 있는 키워드이므로 SEO 선점이 쉽습니다.

## 1단계 — 증거 만들기 (1~2개월, 가장 중요)

프레임워크는 글이 아니라 사례로 퍼집니다. Apple Developer Academy @ POSTECH이 실험장입니다.

- 한 팀 또는 한 세션에 HypoLoop을 실제 적용
- 러너들의 가설 템플릿 원본과 델타 회고를 (동의 하에) 수집
- "사례 보고서" 제작 — 이론과 운영 기록은 전파력이 10배 차이

이 사례가 이후 모든 채널의 탄약이 됩니다.

## 2단계 — 국내 전파 (개발자리 트랙)

- 제안 글을 브런치/블로그에 게시 → GeekNews, 커리어리 공유 (교육 방법론은 GeekNews에서 반응 좋은 카테고리)
- Inflearn: "HypoLoop으로 배우는 X" 형태로 기존 강의 포맷에 프레임워크를 얹기 — 강의 자체가 전파 채널
- 컨퍼런스: 인프콘·Let'Swift에 "팀 프로젝트에서 개인은 배우고 있는가" 제목으로 발표 제안 — 사례 보고서가 있으면 채택 확률 상승

## 3단계 — 글로벌 전파 (devkoan 트랙)

- 제안 글 영문판을 Substack "From Zero to Shipped" 시리즈에 게재 → Dev.to에 canonical 크로스포스트
- Hacker News: "Show HN: HypoLoop – a learning framework where failure is required" 형태로 랜딩 페이지 직접 공유
- Reddit: r/ExperiencedDevs(멘토링 관점), r/learnprogramming(학습자 관점)에 각각 다른 각도로 포스팅 — 쌓아둔 카르마 회수 시점

## 4단계 — 도구화가 최강의 전파

사람들은 프레임워크를 읽지 않고 씁니다.

- 가설 템플릿 + 챌린지 트리 캔버스를 Notion 템플릿으로 무료 배포 — 복제 수 자체가 전파 지표
- 심화 버전 "HypoLoop 운영 워크북"은 Gumroad 제품으로 (vibe coding workbook과 같은 라인)
- 장기 옵션: Distill과 철학이 겹치므로(close-with-artifact ↔ 가설-델타 기록) HypoLoop 사이클 관리 미니 앱

## 순서 요약

홈 → Academy 실증 → 사례 보고서 → 국내 글/발표 → 영문 크로스포스트 → Notion 템플릿 → Gumroad

## 핵심 원칙

사례가 생기기 전에 대대적으로 알리지 말 것. 프레임워크 시장에서 신뢰의 단위는 "적용 후기"입니다.
