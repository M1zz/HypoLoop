> 답이 아니라 질문에서 시작합니다. 질문이 학습의 엔진입니다.
> 실패를 허용하지 않습니다. 요구합니다.
> 팀 프로젝트 안에서 개인의 학습을 의도적으로 설계하는 학습 프레임워크.

## 프레임워크 한 장 요약

1. **학습이 목적** — 팀 산출물은 수단. 충돌 시 개인 학습 우선.
2. **질문이 엔진** — 질문→가설→탐구→평가. 답을 찾기 전에 먼저 묻는다. 질문이 가설을 낳고, 가설이 깨지는 순간이 최대 학습 지점.
3. **구조는 프랙탈** — 팀 층과 개인 층이 같은 사이클로 순환. 개인 챌린지는 팀 챌린지에 중첩.
4. **실패는 요구됨** — 생산적 실패 + 심리적 안전. 깨지지 않는 가설은 경고 신호.
5. **혈통이 진정성** — 계층 트리가 모든 작은 챌린지를 궁극의 진짜 문제와 연결.

## 질문 → 가설 문장 템플릿

```
질문: 나는 [대상 X]가 왜/어떻게 [이렇게 되는지] 정말 궁금하다.
가설: 나는 지금 X를 [이렇게 이해/예상]한다.
검증: [검증 행동 Y]를 해보면 이 이해가 맞는지 드러날 것이다.
학습: 이 가설이 깨지면 나는 [알게 될 것]을 알게 된다.
```

동료 검증 질문: "이 가설이 깨지면, 너는 뭘 알게 되는데?"
