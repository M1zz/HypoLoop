## 프레임워크 한 장 요약

1. **학습이 목적** — 팀 산출물은 수단. 충돌 시 개인 학습 우선.
2. **가설이 엔진** — 가설→질문→탐구→평가. 가설이 깨지는 순간이 최대 학습 지점.
3. **구조는 프랙탈** — 팀 층과 개인 층이 같은 사이클로 순환. 개인 챌린지는 팀 챌린지에 중첩.
4. **실패는 요구됨** — 생산적 실패 + 심리적 안전. 깨지지 않는 가설은 경고 신호.
5. **혈통이 진정성** — 계층 트리가 모든 작은 챌린지를 궁극의 진짜 문제와 연결.
